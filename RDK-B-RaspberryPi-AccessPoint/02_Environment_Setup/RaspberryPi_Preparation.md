# Raspberry Pi Preparation

## Overview

The target hardware platform selected for this project is the **Raspberry Pi 4 Model B (64-bit)**. This board serves as the embedded device that runs the custom Yocto Linux image and functions as a Wi-Fi Access Point.

Before flashing the custom image, the Raspberry Pi hardware must be prepared properly. This includes selecting suitable hardware components, verifying compatibility, and ensuring that the board is ready for booting.

---

# Why Raspberry Pi 4?

The Raspberry Pi 4 was chosen because it provides sufficient processing power and networking capabilities required to implement an embedded Wi-Fi Access Point.

Advantages include:

- Quad-Core ARM Cortex-A72 Processor
- 64-bit ARM Architecture
- Built-in Wi-Fi
- Built-in Bluetooth
- Gigabit Ethernet
- USB 3.0 Support
- Large Community Support
- Official Yocto BSP Support

These features make it an excellent platform for embedded Linux and networking projects.

# MicroSD Card Selection

The operating system image is stored on a MicroSD card.

Recommended Specifications:

| Parameter | Recommendation |
|------------|---------------|
| Capacity | Minimum 16 GB |
| Recommended | 32 GB |
| Speed Class | Class 10 |
| Interface | UHS-I Preferred |

A faster SD card significantly reduces boot time and improves system responsiveness.

---

# Power Supply

The Raspberry Pi 4 requires a stable power source.

Recommended:

```
5V
3A
USB Type-C Power Adapter
```

Using an underpowered adapter may result in:

- Random reboots
- Boot failures
- USB device instability
- Wi-Fi instability

---

# Ethernet Connection

The Ethernet interface provides internet connectivity to the Raspberry Pi.

In this project:

```
Internet
      │
      ▼
Ethernet Router
      │
      ▼
 Raspberry Pi (eth0)
```

The Ethernet interface acts as the WAN connection.

---

# Wireless Interface

The Raspberry Pi provides an onboard wireless interface.

```
wlan0
```

This interface is configured as:

- Wi-Fi Access Point
- DHCP Server Interface
- Client Network

Wireless clients connect through this interface.

---

# Storage Device

The MicroSD card stores:

- Bootloader
- Linux Kernel
- Device Tree
- Root Filesystem
- Custom Configuration Files

All components generated by the Yocto build are written to the SD card.

# LED Indicators

The Raspberry Pi provides onboard LEDs that help determine the boot status.

### Power LED

Indicates:

- Stable power supply
- Board powered correctly

---

### Activity LED

Indicates:

- SD card access
- Boot activity
- Filesystem operations

Continuous blinking during boot is normal.

---

# Cooling Considerations

During Yocto image execution and networking operations, the processor may generate heat.

Recommended:

- Heat Sink
- Cooling Fan (optional)

This helps prevent thermal throttling during extended operation.

---

# Verify Hardware

Before flashing the image, verify:

- Raspberry Pi board is functioning.
- SD card is detected.
- Ethernet cable is connected.
- Power adapter is working.
- Wi-Fi antenna (onboard) is operational.
