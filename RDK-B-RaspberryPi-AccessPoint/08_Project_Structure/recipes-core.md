# recipes-core

## Introduction

The `recipes-core` directory contains recipes responsible for building the core components of the custom Linux image. In this project, the primary purpose of this directory is to define the complete software image that will be generated by Yocto and deployed onto the Raspberry Pi 4.

Unlike package recipes, which install individual software components, an image recipe combines multiple packages into a single bootable Linux image.

In this project, the `recipes-core` directory contains the custom image recipe:

```text
core-image-rdkb-ap.bb
```

This recipe acts as the entry point for building the final operating system.

---

# Purpose of recipes-core

The `recipes-core` directory is responsible for:

- Defining the custom Linux image.
- Selecting software packages.
- Including project-specific recipes.
- Creating the root filesystem.
- Generating bootable images.
- Producing deployment artifacts.

---

The structure used in this project is shown below.


Only one image recipe was required because the project generates a single custom operating system.

# Image Recipe

The image recipe:


Responsibilities include:

- Selecting the base image.

# Installed Packages

The following packages are included through the image recipe.

```bitbake
IMAGE_INSTALL += " \
hostapd \
dnsmasq \
iw \
iptables \
rdkb-ap-config \
"
```

Each package has a specific role.

---

## hostapd

Purpose:

Transforms the Raspberry Pi into a wireless Access Point.

Provides:

- SSID broadcasting
- WPA2 authentication
- Client association
- Wireless management

---

## dnsmasq

Purpose:

Provides DHCP and DNS services.

Responsibilities:

- IP address assignment
- Gateway information
- DNS forwarding
- Lease management

---

## iw

Purpose:

Wireless interface management.

Used for:

- Viewing wireless devices
- Displaying connected stations
- Monitoring Wi-Fi status

Example command:

```bash
iw dev wlan0 station dump
```

---

## iptables

Purpose:

Provides Linux firewall and Network Address Translation (NAT).

Responsibilities:

- Packet forwarding
- Internet sharing
- NAT
- Routing

Without this package, Internet sharing is not possible.

---

## rdkb-ap-config

Purpose:

Installs all project-specific configuration files.

Includes:

- hostapd.conf
- rdkb-ap.conf
- rdkb-network.sh
- rdkb-network.service

This package integrates all custom project components into the final image.

---

# IMAGE_INSTALL Variable

The most important section of the image recipe is:

```bitbake
IMAGE_INSTALL += " \
hostapd \
dnsmasq \
iw \
iptables \
rdkb-ap-config \
"
```

`IMAGE_INSTALL` tells BitBake which packages should be copied into the generated root filesystem.

During the build:

```text
BitBake

     │

     ▼

IMAGE_INSTALL

     │

     ▼

Dependency Resolution

     │

     ▼

Package Installation

     │

     ▼

Root Filesystem Creation
```

# Image Verification

Before flashing, the generated image was verified.

Mount image.

```bash
sudo losetup -Pf --show verify.wic

sudo mount /dev/loopXp2 /mnt/verify-root
```

Verify installed packages.

Example:

```bash
find /mnt/verify-root -name hostapd
```

```bash
find /mnt/verify-root -name dnsmasq
```

```bash
find /mnt/verify-root -name iptables
```

Expected:

```text
/usr/sbin/hostapd

/usr/bin/dnsmasq

/usr/sbin/iptables
```

---

# Verify Installed Scripts

Check:

```bash
cat /mnt/verify-root/usr/bin/rdkb-network.sh
```

Expected:

Network initialization script.

Display:

```bash
cat /mnt/verify-root/etc/hostapd.conf
```

Display:

The image recipe included:

- `hostapd`
- `dnsmasq`
- `iw`
- `iptables`

After updating the recipe, the image was rebuilt using BitBake. The generated image was then mounted and inspected to verify that all packages, configuration files, scripts, and service files were correctly installed before flashing it onto the Raspberry Pi.

---

# Lessons Learned

Working with the `recipes-core` directory provided several important insights:

- The image recipe defines the complete operating system.
- Every required package must be listed in `IMAGE_INSTALL`.
- Missing packages result in missing functionality.
- Image verification before flashing helps detect packaging errors.
- Keeping the image recipe simple improves maintainability.
- Project-specific packages should be grouped into custom recipes instead of directly copying files into the image recipe.

# Conclusion

The `recipes-core` directory is responsible for defining the complete Linux operating system generated by Yocto. Through the `core-image-rdkb-ap.bb` image recipe, all required networking packages, custom configuration files, and project-specific components were integrated into a single bootable image. This modular approach simplified system customization, ensured reproducible builds, and provided a reliable foundation for the Raspberry Pi-based RDK-B Access Point implementation.
